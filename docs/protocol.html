EquityMA: A Browser-Based Tool for Equity-Stratified Meta-Analysis Using the PROGRESS-Plus Framework

Protocol
Mahmood Alhusseini
Independent Researcher & Meta-Analysis Methods Developer

Abstract

Background: Despite widespread endorsement of the PROGRESS-Plus framework for equity-sensitive evidence synthesis, no purpose-built browser tool exists to perform equity-stratified meta-analysis. Investigators must rely on general-purpose software that lacks equity-specific workflows, PRISMA-Equity (PRISMA-E) compliance checking, and equity gap visualisation.

Objectives: To build EquityMA, the first browser-based, offline-capable tool that enables researchers to (1) enter study-level effect sizes with PROGRESS-Plus subgroup data, (2) pool effects within equity strata using DerSimonian-Laird random-effects models, (3) test for equity-dimension interactions with the Q-between statistic, (4) visualise stratified forest plots and equity gap charts, (5) assess reporting completeness with an auto-fill PRISMA-Equity checklist, and (6) export results to CSV/JSON.

Methods: Single-file HTML/JS application; DerSimonian-Laird inverse-variance weighting; Q-between interaction test; Canvas-based visualisations; localStorage persistence; no server required.

Registration: EQM-2026-001

1. Background and Rationale

Health equity in evidence synthesis requires systematic examination of whether treatment effects vary across socially-defined subgroups. The PROGRESS-Plus framework — covering Place of residence, Race/ethnicity/culture/language, Occupation, Gender/sex, Religion, Education, Socioeconomic status, and Social capital, plus additional "Plus" dimensions — provides a widely-used taxonomy for equity-relevant stratifiers.

Despite this framework being recommended by Cochrane, PRISMA-Equity 2012, and the Campbell & Cochrane Equity Methods Group, no browser-native tool provides an integrated workflow for equity-stratified meta-analysis. Existing general-purpose meta-analysis software (RevMan, metafor, R meta) requires manual adaptation and provides no PRISMA-E checklist integration or equity gap visualisation.

The gap is consequential: systematic reviews that fail to report equity analyses perpetuate evidence gaps that disproportionately affect marginalised populations.

2. Objectives

  1. Implement DerSimonian-Laird random-effects pooling within each PROGRESS-Plus equity stratum.
  2. Compute Q-between interaction statistics to formally test whether pooled effects differ across equity groups.
  3. Render interactive stratified forest plots using HTML5 Canvas.
  4. Display equity gap charts showing pooled effect differentials between strata.
  5. Provide a reporting heatmap summarising PROGRESS-Plus coverage across included studies.
  6. Implement an auto-fill PRISMA-Equity 2012 checklist with 12 reporting items.
  7. Support CSV import/export and JSON state persistence for reproducibility.

3. Methods

3.1 Statistical Methods

Within-stratum pooling: For each equity dimension stratum, effect sizes are pooled using DerSimonian-Laird (DL) random-effects meta-analysis. Log transformation is applied for ratio outcomes (RR, OR, HR). The DL tau-squared estimator is computed as:

tau2 = max(0, (Q - (k-1)) / (W - W2/W))

where Q is Cochran's heterogeneity statistic, k is the number of studies in the stratum, W is the sum of inverse-variance weights, and W2 is the sum of squared weights.

Equity interaction test: The Q-between statistic tests whether pooled estimates differ across strata within a PROGRESS-Plus dimension. A chi-squared test with (s-1) degrees of freedom (s = number of strata) is used, with p < 0.05 indicating a statistically significant equity interaction.

Effect types supported: Relative Risk (RR), Odds Ratio (OR), Hazard Ratio (HR), Mean Difference (MD), Standardised Mean Difference (SMD).

3.3 Visualisation

All plots are rendered using HTML5 Canvas for offline capability and cross-browser compatibility:

  • Stratified forest plot: Study-level and pooled estimates per stratum, with logarithmic x-axis for ratio outcomes.
  • Equity gap chart: Bar chart of pooled effect sizes by stratum, with colour-coded significance indicators.
  • Coverage heatmap: Grid of studies (rows) by PROGRESS-Plus dimensions (columns), showing reporting completeness.

3.4 PRISMA-Equity Checklist

The tool implements the 12-item PRISMA-Equity 2012 extension checklist. Items are auto-populated where evidence is available from the entered dataset (e.g., equity dimensions reported, subgroup analyses performed). Manual confirmation is required for items requiring human judgement (e.g., a priori specification of equity analyses, author equity expertise).

3.5 Software Architecture

EquityMA is implemented as a single self-contained HTML file (<1,800 lines) requiring no server, no installation, and no internet connection after initial download. State is persisted to browser localStorage. The application is distributed under the MIT licence.

4. Outcomes

  • Primary: Stratified pooled effect sizes with 95% confidence intervals per equity dimension.
  • Secondary: Q-between interaction p-values; I-squared heterogeneity within strata; equity gap magnitude; PRISMA-E compliance score.
  • Exploratory: Reporting completeness heatmap across PROGRESS-Plus dimensions.

5. Example Datasets

Three built-in example datasets are provided for demonstration and validation:

  1. Statins by SES — 6 cardiovascular RCTs with socioeconomic stratification (WOSCOPS, AFCAPS, ASCOT, CARDS, JUPITER, TNT).
  2. Vaccines by Country Income — 15 vaccine trials stratified by high-income vs. low-income country setting (BCG, RTS,S, OCV trials).
  3. Antihypertensives by Sex — 6 hypertension RCTs stratified by sex (ALLHAT, ACCOMPLISH, LIFE, VALUE, HOPE, ONTARGET).

6. Dissemination

The tool will be published open-access on GitHub Pages and described in a companion results paper. Source code and test fixtures are deposited in the GitHub repository.
