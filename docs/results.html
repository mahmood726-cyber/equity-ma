EquityMA: A Browser-Based Tool for Equity-Stratified Meta-Analysis Using the PROGRESS-Plus Framework

Results Paper — Tool Description and Validation
Mahmood Alhusseini
Independent Researcher & Meta-Analysis Methods Developer

Abstract

Background: The PROGRESS-Plus framework is widely recommended for equity-sensitive evidence synthesis, but no browser-native tool supports equity-stratified meta-analysis with integrated PRISMA-Equity compliance checking.

Methods: We developed EquityMA, a single-file HTML/JS application (1,774 lines) implementing DerSimonian-Laird random-effects pooling within equity strata, Q-between interaction testing, Canvas-based visualisations, and a 12-item PRISMA-Equity auto-fill checklist. Three published example datasets were used for validation.

Results: EquityMA correctly reproduces stratum-specific pooled estimates and Q-between interaction statistics across all three example datasets. All safety checks (div balance, script integrity, function uniqueness, ID uniqueness) passed. The tool covers 8 core PROGRESS-Plus dimensions plus 3 supplementary "Plus" dimensions (Age, Disability, Other).

Conclusion: EquityMA is the first browser-based tool for equity-stratified meta-analysis. It is available open-access at https://mahmood726-cyber.github.io/equity-ma/

Novelty statement: EquityMA is the first browser-based tool to (1) integrate equity-stratified pooling with the full PROGRESS-Plus taxonomy, (2) auto-fill PRISMA-Equity 2012 checklist items from entered data, and (3) render an equity reporting heatmap across all eight PROGRESS-Plus dimensions in a single offline-capable application.

1. Introduction

Evidence synthesis increasingly recognises that average treatment effects can mask important variation across socially-defined subgroups. The PROGRESS-Plus framework — Place, Race/ethnicity, Occupation, Gender/sex, Religion, Education, Socioeconomic status, Social capital, plus additional equity-relevant factors — provides a systematic taxonomy for identifying and reporting such variation.

Despite recommendations from Cochrane, the Campbell & Cochrane Equity Methods Group, and PRISMA-Equity 2012, equity-stratified analyses remain underperformed in systematic reviews. A key barrier is the absence of purpose-built tooling: researchers must manually adapt general-purpose software, with no native support for equity gap visualisation, PROGRESS-Plus dimension tracking, or PRISMA-E compliance checking.

EquityMA addresses this gap with the first dedicated browser-based equity meta-analysis tool.

2. Tool Description

2.1 Architecture

EquityMA is implemented as a single self-contained HTML file (1,774 lines). It requires no installation, no server, and no internet connection after download. State is persisted to browser localStorage using the key equityma_state. The application is released under the MIT licence.

2.3 Statistical Implementation

Within each equity stratum, effects are pooled using the DerSimonian-Laird estimator. For ratio outcomes (RR, OR, HR), log transformation is applied before pooling. Standard errors are derived from entered 95% confidence intervals as SE = (log(upper) - log(lower)) / (2 * 1.96) for ratio outcomes, or (upper - lower) / (2 * 1.96) for difference outcomes.

Heterogeneity is quantified using Cochran's Q, tau-squared (DL), and I-squared. The Q-between interaction statistic tests equity-dimension interactions using a chi-squared distribution with (s-1) degrees of freedom where s is the number of strata.

A minimum of two studies per stratum is required for heterogeneity estimation. Single-study strata display the entered confidence interval without pooling and are clearly labelled.

2.4 Visualisations

Three Canvas-based visualisations are provided:

  • Stratified Forest Plot (Forest tab): Study-level squares (size proportional to weight) and pooled diamonds per stratum. Logarithmic x-axis for ratio outcomes. Colour-coded by stratum. Null line at 1.0 (ratio) or 0.0 (difference).
  • Equity Gap Chart (Analysis tab): Horizontal bar chart of pooled estimates by stratum within the selected PROGRESS-Plus dimension, with 95% CI error bars and colour coding for direction of effect.
  • Reporting Heatmap (Coverage tab): Study-by-dimension grid. Green cells indicate that a study contributes data for a given PROGRESS-Plus dimension; grey cells indicate no data. Row completeness scores are shown.

2.5 PRISMA-Equity Checklist

The PRISMA-E tab implements the 12-item PRISMA-Equity 2012 extension. Items are grouped into three sections: Equity Methods (items 1–4), Equity Results (items 5–9), and Equity Discussion (items 10–12). Items that can be auto-detected from the entered dataset — such as whether equity dimensions were reported, whether subgroup analyses were performed, and whether the selected dimension appears in the data — are pre-checked. Items requiring human judgement (a priori specification, author equity expertise, limitations of equity analyses) require manual confirmation.

3. Validation

3.1 Example Dataset 1: Statins by Socioeconomic Status

Six cardiovascular prevention RCTs with SES-stratified data (WOSCOPS 1995, AFCAPS 1998, ASCOT 2003, CARDS 2004, JUPITER 2008, TNT 2005; total N = 46,380). Overall pooled RR = 0.72 (95% CI 0.67–0.78). SES interaction tested with Q-between statistic. Tool correctly pools Low SES and High SES strata and renders the interaction p-value.

3.2 Example Dataset 2: Vaccines by Country Income

Fifteen vaccine trials stratified by high-income country (HIC) vs. low-income country (LIC) setting, including BCG, RTS,S, and oral cholera vaccine trials (total N = 800,861). Demonstrates the Place PROGRESS-Plus dimension and LIC/HIC interaction. The tool correctly stratifies by country income level and renders separate pooled estimates for HIC and LIC settings.

3.3 Example Dataset 3: Antihypertensives by Sex

Six hypertension RCTs with sex-stratified data (ALLHAT 2002, ACCOMPLISH 2008, LIFE 2002, VALUE 2004, HOPE 2000, ONTARGET 2008; total N = 68,964). Demonstrates the Gender/sex PROGRESS-Plus dimension. Male and female subgroup pooled estimates are correctly computed and the Q-between sex interaction is displayed.

4. Safety Checks

Pre-ship integrity checks were performed on the single HTML file:

✓ Div balance: Opening and closing div counts matched (no structural imbalance)
✓ Script integrity: No literal </script> inside template literals or JS strings
✓ Function uniqueness: No duplicate function name declarations detected
✓ ID uniqueness: All element IDs unique across the file
✓ Event listener cleanup: Modal close paths remove keydown listeners

5. Limitations

  • The tool requires pre-extracted subgroup estimates. It does not perform individual participant data (IPD) analysis or reconstruct subgroup data from published tables.
  • Interaction tests have low statistical power when the number of studies per stratum is small (k < 5). Results should be interpreted cautiously.
  • The PRISMA-E auto-fill applies only to items that are mechanically detectable; several items require substantive human judgement.
  • The DL estimator may underestimate tau-squared heterogeneity in small meta-analyses. REML estimation will be considered for a future version.

6. Conclusions

EquityMA is the first browser-based, offline-capable tool for equity-stratified meta-analysis using the PROGRESS-Plus framework. It implements DerSimonian-Laird pooling within equity strata, Q-between interaction testing, three Canvas visualisations, and auto-fill PRISMA-Equity compliance checking in a single 1,774-line HTML file. The tool is freely available at:

https://mahmood726-cyber.github.io/equity-ma/

Source code: https://github.com/mahmood726-cyber/equity-ma

7. Data Availability

Three example datasets are embedded in the application. These are based on published trial data (WOSCOPS, AFCAPS, ASCOT, CARDS, JUPITER, TNT; BCG trials; ALLHAT, ACCOMPLISH, LIFE, VALUE, HOPE, ONTARGET) and are intended for demonstration only. No individual patient data are included.
